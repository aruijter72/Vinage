Vinage — Toegangscodes Beheren

Stap-voor-stap handleiding voor het aanmaken, intrekken en beheren van beta-toegang

📋 Hoe werkt het systeem?

Codes worden opgeslagen in Firestore. Jij beheert ze vanuit de Firebase Console — de app leest ze bij elke invoer live op. Dat betekent:

ActieWat je doetEffect
Nieuwe tester geven Stuur de codecode (bijv. VINAGE-BETA-NAAM) Direct actief
Iemand blokkeren UID toevoegen aan revokedUsers Meteen geblokkeerd
Hele code uitschakelen active → false Niemand kan meer in
Plan terugzetten users/{uid} → plan: "free" Direct downgrade
➕ Een nieuwe toegangscode aanmaken
1

Ga naar de Firebase Console

Open console.firebase.google.com → selecteer project vinage-85fd8 → klik op Firestore Database in het linkermenu.

2

Maak de collectie betaCodes aan (eerste keer)

Klik op + Collectie aanmaken en typ als naam: betaCodes

💡 Let op De collectie bestaat al als je eerder een code hebt aangemaakt. Sla deze stap dan over.
3

Maak een nieuw document aan

Klik op + Document toevoegen. Gebruik als Document-ID de toegangscode zelf — dit is ook wat de tester intypt in de app:

betaCodes › VINAGE-BETA-NAAM

Gebruik per persoon een unieke code zodat je ze apart kunt intrekken, bijv. VINAGE-BETA-PETRA.

4

Voeg de volgende velden toe

🔥 Firestore — betaCodes / VINAGE-BETA-NAAM
active boolean true
plan string "verzamelaar" Of "liefhebber" voor beperktere toegang
revokedUsers array [ ] (leeg laten)
label string "Petra - testaccount" Ter herinnering voor jezelf, de app gebruikt dit niet

Klik op Opslaan. De code is meteen actief.

5

Stuur de code naar je tester

De tester opent Vinage → Instellingen → Over & juridisch → tikt 5× op de versietekst onderaan → voert de code in → tikt OK.

💡 Vereiste: ingelogd zijn De tester moet ingelogd zijn in de app voordat ze een code kunnen invoeren. De code wordt anders niet geaccepteerd.
🚫 Toegang intrekken voor één persoon
1

Zoek de UID van de gebruiker

Ga in Firebase Console naar Authentication → Users. Zoek de persoon op e-mailadres. De UID staat in de kolom User UID:

🔐 Authentication — Users
👤 petra@email.nl UID: xK9mPqR2nTw...
Kopieer de volledige UID — je hebt hem zo meteen nodig.
xK9mPqR2nTwVy8ZjLcFhA4bGe3sD7iNu
2

Voeg de UID toe aan revokedUsers

Ga naar Firestore → betaCodes → {de code die ze hebben} → klik op het potloodpictogram naast revokedUsers → voeg de UID toe als nieuw array-item:

🔥 Firestore — betaCodes / VINAGE-BETA-PETRA
active boolean true
revokedUsers array [ "xK9mPqR2nTwVy8ZjLcFhA4bGe3sD7iNu" ]

Sla op. De gebruiker kan de code niet meer opnieuw invoeren.

3

Zet hun plan terug naar gratis

Ga naar Firestore → users → {hun UID} → wijzig het veld plan naar "free":

🔥 Firestore — users / xK9mPqR2nTw...
plan string "free" Was: "verzamelaar"
email string "petra@email.nl"

De app pikt dit direct op — de volgende keer dat de app opent, zit de gebruiker op het gratis plan. Geen actie vereist van de gebruiker.

⚡ Een code volledig uitschakelen

Gebruik dit als een code gelekt is, of als je alle testers van een specifieke code wilt stoppen in één keer.

1

Ga naar het code-document in Firestore

Firestore → betaCodes → klik op de code.

2

Zet active op false

🔥 Firestore — betaCodes / VINAGE-BETA-NAAM
active boolean false

Niemand kan de code meer gebruiken — ook bestaande gebruikers die hem al hadden kunnen hem niet opnieuw invoeren. Vergeet niet ook hun plan terug te zetten als dat nodig is (zie stap hierboven).

⚡ Snelreferentie — wat moet ik doen?
SituatieActieWaar
Nieuwe tester toevoegen Nieuw document in betaCodes, stuur code Firestore → betaCodes
Eén persoon blokkeren UID in revokedUsers + plan → "free" Firestore → betaCodes + users
Alle gebruikers van één code stoppen active → false Firestore → betaCodes
Iemand tijdelijk downgraden plan → "liefhebber" of "free" Firestore → users → {uid}
UID van een gebruiker opzoeken Authentication → Users → zoek op e-mail Firebase → Authentication
Welke code heeft iemand gebruikt? Veld betaCode in het user-document Firestore → users → {uid}
